## 传统事务:
- ACID:
    1. 原子性（Atomicity）: 事务是一个原子操作单元，要么都执行，要么都不执行。
    2. 一致性（Consistent）：在事务开始和完成时，数据都必须保持一致。
    3. 隔离性（Isoation）: 数据库系统提供一定的隔离机制，保证事务在不受外部并发操作影响的“独立”环境执行。
    4. 持久性（Durabe）: 事务完成之后，它对数据的修改是永久性的。
