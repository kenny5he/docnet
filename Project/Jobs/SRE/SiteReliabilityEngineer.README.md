# Site Reliability Engineer (站点可靠性工程师)

## 能力要求
1. 研发能力
2. 问题分析能力
3. 项目管理能力

## 工作职责
1. 解决系统出现的问题，并制定措施防止其重现
2. 提升线上服务的可用性/可扩展性，优化低延迟/高效率的软件系统
3. 研发新的规范标准、运营方法论
4. 参与业务运维，解决现网问题、发布软件变更、设计容量规划、评估系统性能等，并通过软件工程方法建立相应系统解决运维问题，保障业务运行稳定性

## 工作目标
SRE的工作目标是确保系统的服务等级指标，以及通过开发自动化运维工具来降低人工操作成本，提高运维效率和质量；而运维更强调保障业务功能指标。
