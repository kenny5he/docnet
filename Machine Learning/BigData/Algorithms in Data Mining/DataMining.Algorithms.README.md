# Algorithms in Data Mining (数据挖掘算法)
- 根据是否有类别标签区分可划分为 有监督学习、无监督学习、强化学习
1. 基本任务:
   1. 分类预测 (低收入群体、高收入群体)
   2. 关联规则 (购买手机电子产品 会购买 贴膜、保护壳、耳机)
   3. 聚类分析 ()
   4. 时间序列 (打印机客户 可能每3月购买墨盒)
2. 步骤:
   1. Define Problem - 问题定义
   2. Data Collection - 数据采集/存储
   3. Data preparation - 数据预处理
   4. Data Modelling - 数据模型构建
   5. Interpretation/Evaluation - 数据模型解释与评价
   6. Implement/Deploy model - 实施部署模型
3. 常见误区:
   1. No Free Lunch (数据算法繁多，没有一个算法是绝对优于其它算法的，也没有一种参数设置能对所有问题都能取得良好效果)(一般越简单越好 - 奥卡姆剃刀原理)
   2. 无规律可言的 (彩票)
   3. 关联关系 (属性之间的关联关系需要背景支撑)
   4. 幸存者偏见 (二战存活战机受攻击点的位置，并非发动机、油箱等关键位置)
   5. 数据结论是否遗漏关键属性(时间维度？)
4. 数据挖掘所需基本条件:
   1. 注重数据来源，数据真实性
   2. 深入案例应用的背景知识
   3. 数据使用中的隐私保护
5. 属性类型:
   1. 名义型(Nominal): 
       1. 颜色 Color
   2. 有序型(Ordinal): 
       1. 考试成绩: 优、良、差
   3. 区间型(Interval): 
       1. 身高(150~210)
   4. 比率型(Ratio):
       1. 长度: 米、厘米， 重量: 千克、克
6. 数值属性:
   1. 类别属性: (可有序/无序)、(二元: 对称/不对称(体检阳性/阴性))
   2. 数值属性: 区间标度属性、比率标度属性
   3. 离散、连续属性: 
7. 数据结构重要特征:
   1. 维度
   2. 稀疏
   3. 分辨率
   4. 分布 (中心性分散)
