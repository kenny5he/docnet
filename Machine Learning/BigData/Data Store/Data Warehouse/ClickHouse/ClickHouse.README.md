# ClickHouse
1. 列式存储数据库
2. 特点:
    1. 真正的面向列的DBMS
    2. 数据高效压缩
    3. 磁盘存储的数据
    4. 多核并行处理
    5. 在多个服务器上分布式处理
    6. SQL语法支持
    7. 向量化引擎
    8. 实时数据更新
    9. 索引
    10. 适合在线查询
    11. 支持近似预估计算
    12. 支持嵌套的数据结构, 支持数组作为数据类型
    13. 支持限制查询复杂性以及配额
    14. 复制数据复制和对数据完整性的支持
3. 缺点:
    1. 不支持事物。
    2. 不支持Update/Delete操作。
    3. 不支持高并发，官方建议qps为100，(尽量做1000条以上批量的写入，ClickHouse底层会不断的做异步的数据合并，会影响查询性能)
    4. 支持有限操作系统。(ubuntu,centos)
4. 应用场景:
    1. 存储数据和统计数据使用
    2. 用户行为数据记录和分析工作。
    3. 用于广告网络和RTB,电子商务的用户行为分析。
    4. 信息安全里面的日志分析
    5. 检测和遥感信息的挖掘。
