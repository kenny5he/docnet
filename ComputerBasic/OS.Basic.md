# 计算机原理
1. OS的四类资源：处理器、存储器、I/O设备以及信息（数据和程序）
2. OS的主要功能：
    1. 处理机管理：用于分配和控制处理机
    2. 存储器管理：主要负责内存的分配与回收
    3. I/O设备管理，负责 I/O 设备的分配(回收)与操作
    4. 文件管理，负责文件的存取、共享和保护

## 处理器
1. 功能
    1. 创建和撤销进程（线程）
    2. 对诸进程（线程）的运行进行协调
    3. 实现进程（线程）之间的信息交换
    4. 按照一定算法把处理机分配给进程（线程）
    5. 进程控制
    6. 进程同步
    7. 进程通信
    8. 调度
2. 进程同步
    1. 进程互斥方式：诸进程在对临界资源进行访问时，应采用互斥方式
    2. 进程同步方式：在相互合作完成共同任务的诸进程间，由同步机构对它们的执行次序加以协调

# 操作系统
## 操作系统的基本特性
### 并发 (Concurrence)
1. 并行与并发
    1. 并行 Parallel: 
        - 两个或多个事件在同一时刻发生，并行发生在多个CPU或CPU和I/O设备之间
        - 例：早上一边微波炉在做早饭，一边洗衣机在洗衣服，一边在阳台上温暖的阳光下阅读
    2. 并发 Concurrence:
        - 两个或多个事件在同一时间间隔内发生
        - 例：上午做三件事情，修订计算机专业应用缩写，修订操作系统原理课件、写工作日记
2. 引入进程 (Process)
    1. 程序是静态实体，在多道程序体系中，程序是不能独立运行的，更不能和其他程序并发执行。引入进程的目的，使多个程序（进程）并发执行。
    2. 在为引入进程的系统中，在属于同一个应用程序的计算程序和IO程序之间，只能顺序执行。但在引入进程后，若分别为计算程序和IO程序各建立一个进程，则这两个进程便可并发执行。
    3. 由于在系统中具备使计算程序和IO程序同时运行的硬件条件，因而可将系统中的CPU和IO设备并行工作
    4. 在内存存放多个用户程序，分别建立进程后，这些进程可以并发执行实现多道程序执行。
    5. 进程是指在系统中能独立运行并作为资源分配的基本单位
    6. 多个进程之间可以并发执行和交换信息
    7. 进程和并发是现代OS的重要基本概念，也是OS运行的基础。
### 共享 (Sharding)
1. 共享：
   - 系统中的资源可供内存中多个并发执行的进程（线程）共同使用
2. 资源共享方式：
    - 互斥共享方式
    - 同时访问方式
### 虚拟 (Virtual)
- 虚拟，通过某种技术把一个物理实体变为若干个逻辑上的对应物
- 用于实现虚拟的技术为虚拟技术
- 在OS中利用两种方式实现虚拟技术，即时分复用技术和空分复用技术
1. 时分复用技术
    1. 时分复用：即分时使用方式
    2. 在计算机领域中，广泛利用该技术来实现虚拟处理机、虚拟设备等，以提高资源利用率
2. 空分复用技术： 提供存储空间的利用率
    1. 虚拟磁盘技术:
        
    2. 虚拟存储器技术：
        1. 空分复用则是利用存储器的空间空间来存放其他程序，以提高内存的利用率
        2. 单纯空分复用存储器只能提高内存的利用率，并不能实现在逻辑上扩大存储器容量的功能，必须引入虚拟存储器技术才能达到此目的。
### 异步 (Asynchronism)
- 在多道程序环境下允许多个进程并发执行，但只有进程获得所需资源后方执行。在单处理机环境下，由于系统中只有一台处理机，因而每次只能运行一个进程执行，其余进程只能等待。
- 由于资源等因素限制，进程的执行是以 "停停走走"的方式执行。
- 进程是以人们不可预知的速度向前推进，此即进程的异步性。
