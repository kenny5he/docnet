## 进程
1. 进程的定义
    1. 使参与并发执行的每个程序（含数据）都能独立地运行，在OS系统中必须为之配置进程控制块PCB(Process Control Block)
    2. 系统利用PCB来描述进程基本情况和活动过程，进而控制（Control）和管理(Manage)进程。由程序段、相关数据段和PCB使够吃进程实体（进程映像）
    3. 进程是程序的一次执行running
    4. 进程是一个程序 program 及其数据 data在处理机 processor 上顺序执行 execute 时所发生的活动
    5. 进程是具有独立功能的程序在一个数据集合上运行的过程，它是系统进行资源分配和调度的一个独立单位。
2. 进程的特征
   1. 动态性
      1. 进程的实质是进程实体的执行过程（动态性Dynamic 是进程的最基本的特征）
      2. 进程process由创建 Create而产生，由调度Scheduling而执行，由撤销 Withdraw而消亡，进程实体有一定的生命周期 Life Cycle
      3. 程序Program只是一组有序指令Instruction的集合，并存放于某种介质上，其本身并不具有活动的含义，而是静态Static的
   2. 并发性
      1. 多个进程实体同存于内存memory中，且能在一段时间内同时运行
      2. 引入进程的目的是为进程实体能和其它进程实体并发执行
      3. 并发性是进程的另一重要特性
   3. 独立性
      1. 进程实体是一个独立运行的基本单位，独立获得资源和独立接受调度的基本单位
      2. 未建立PCB程序都不能作为一个独立的单位参与运行
   4. 异步性(Asynchronization)
      1. 进程是按异步方式运行的，即按各自独立的、不可预知的速度向前推进
      2. 程序若参与并发执行，会产生结果不可再现性，未使进程在并发执行时结果可再现，在OS系统中引入进程概念，并且配置相应的进程同步机制 Synchronization mechanism
3. 进程的基本状态与转换
   - 由于多个进程在并发执行共享系统资源，致使它们在运行过程中呈现间断性的运行规律，进程在生命周期内可能具有多种状态
   - 为满足进程控制块对数据及操作的完整性要求及增强管理的灵活性，通常引入两种常见状态（创建状态、终止状态）
   - 进程处于创建状态与终止状态只有一次
   1. 就绪状态 Ready
      1. 进程已处于准备好运行的桩体，即进程已分配到CPU以外必要资源后，只要在获得CPU，便可立即运行
      2. 如果系统中有许多处于就绪状态的进程，通常将它们按一定的策略排成一个队列，称该队列为就绪队列
   2. 执行状态 Running
      1. 进程已获得CPU，其程序正在执行的状态
      2. 对任何一个时刻而言，但处理机系统中，只有一个进程处于执行状态
      3. 而在多处理器系统中，则有多个进程处于执行状态
   3. 阻塞状态 Blocked
      1. 正在执行的进程由于发生某事件暂时无法继续执行时的状态，亦即进程的执行受到阻塞，此时引起进程调度，OS处理机分配另一个就绪进程，而让受阻塞进程处于一个暂停状态，把这个暂停状态称为阻塞状态
      2. 通常系统将处于阻塞状态的进程排成一个队列，称该队列为阻塞队列
      3. 阻塞队列恢复运行 需 重新排队
   4. 创建状态 New
      1. 创建进程的步骤
         1. 由进程申请一个空白 PCB
         2. 向PCB中填写用于控制管理进程的信息
         3. 为进程分配运行时所需的资源
         4. 把该进程转入就绪状态并插入就绪队列
         - 注: 如果进程所需的资源尚不能得到满足，比如系统尚无足够的内存使进程无法装入其中，即创建工作尚未完成，进程不能被调度执行，其所处的状态就是创建状态
   5. 终止状态 Terminated
      1. 进程终止步骤
         1. 等待OS进行善后处理
         2. 将其PCB清零，将PCB空间返还系统
      2. 进程进入终止状态的几种情况
         1. 到达自然结束点
         2. 出现无法克服的错误
         3. 被OS终结
         4. 被其他有终止权的进程所终结
   
4. 挂起操作和进程状态转换
   1. 进程状态转换
      1. 就绪状态 -> 运行状态 （进程调度）
      2. 运行状态 -> 阻塞桩体 (I/O请求)
      3. 阻塞状态 -> 就绪状态 （I/O完成）
      4. 执行状态 -> 就绪状态 （时间分片完）

6. 进程同步
   1. 在OS引入进程后，虽然提高资源利用率和系统吞吐量，但由于进程的异步性，也会给系统造成混乱，尤其是他们争用临界资源时。
   2. 主要任务： 对多个相关进程在执行次序上进行协调，以使并发执行的诸进程之间能有效地共享资源和互相合作，从而使程序的执行具有可再现性。
   3. 制约关系：
       1. 间接相互制约关系
       2. 直接相互制约关系
